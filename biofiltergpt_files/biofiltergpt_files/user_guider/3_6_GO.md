## ðŸ§¬ What Is the Gene Ontology in Biofilter3R?

The **Gene Ontology (GO)** provides a structured vocabulary to describe **molecular functions**, **biological processes**, and **cellular components** of genes and gene products.

In Biofilter3R, GO terms are represented using a combination of **dedicated models** and the **shared entity infrastructure**, allowing:

- Rich navigation through the GO DAG (directed acyclic graph)
- Entity-level integration (genes, proteins, etc.)
- Flexible storage of relationships, annotations, and namespaces

## ðŸ” `GORelation`

This table captures the **graph structure** of the Gene Ontology, where terms can be related hierarchically:

| Field | Description |
| --- | --- |
| `parent_id` | ID of the parent term (`GOMaster.id`) |
| `child_id` | ID of the child term (`GOMaster.id`) |
| `relation_type` | Relationship type (`is_a`, `part_of`, etc.) |

The system supports **multiple relation types**, representing the DAG structure of the ontology. Examples:

| Relation Type | Meaning |
| --- | --- |
| `is_a` | The child is a more specific subtype of the parent |
| `part_of` | The child is a component or step within the parent |

## ðŸ’¡ Design Highlights

- **Entity-based modeling**: Each GO term is treated as an `Entity`, ensuring standardization and relationship compatibility
- **Fully navigable DAG**: The `GORelation` table enables upward and downward traversal in the ontology graph

- **Future-ready**: Easily extendable to include additional metadata, evidence, and external cross-references
